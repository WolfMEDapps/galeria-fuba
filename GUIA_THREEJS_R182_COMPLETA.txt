ERRORES COMETIDOS EN SESIONES ANTERIORES Y SUS SOLUCIONES
================================================================================

ERROR 1 — Tocar cosas no pedidas
  Solución: Tocar ÚNICAMENTE lo que se pide.

ERROR 2 — MeshPhysicalMaterial → MeshLambertMaterial sin permiso
  Solución: Nunca cambiar tipo de material sin permiso explícito.

ERROR 3 — Quitar secciones del panel sin permiso
  Solución: Nunca eliminar controles del panel sin pedido explícito.

ERROR 4 — LIGHT_NAMES en orden incorrecto
  Solución: Extraer orden real del GLB con pygltflib antes de hardcodear.
  El orden correcto es el que devuelve traverse() al recorrer los nodos.

ERROR 5 — ssaoPass.normalMaterial.side no configurado
  Causaba halos blancos en bordes de meshes DoubleSide.
  Solución: ssaoPass.normalMaterial.side = THREE.DoubleSide

ERROR 6 — shadowMap.type no se puede cambiar en runtime
  Solución: Slider de calidad controla shadow.mapSize via _shadowMapSize,
  no shadowMap.type.

ERROR 7 — sceneMeshes.forEach fuera del callback de GALERIA.glb
  Solución: SIEMPRE dentro del callback onLoad, después del traverse.

ERROR 8 — castShadow sin configuración completa
  Solución: Siempre incluir mapSize, bias y normalBias:
    light.shadow.mapSize.set(_shadowMapSize, _shadowMapSize);
    light.shadow.bias=-0.0003;
    light.shadow.normalBias=0.02;

ERROR 9 — Sistema de descarga de main.js con fetch + regex
  RESUELTO en Sesión 4: ya no se intenta modificar main.js en runtime.
  El flujo correcto para persistir valores de código es:
    1. Usuario ajusta panel → presiona COPIAR VALORES
    2. Pega los valores a Claude junto con el main.js actual
    3. Claude aplica los cambios al archivo subido por el usuario
  Para persistir el estado visual completo sin Claude:
    → Usar GUARDAR ESCENA .galeria / CARGAR ESCENA .galeria

ERROR 10 — Trabajar desde versiones desactualizadas del archivo
  Solución: SIEMPRE trabajar desde el archivo que el usuario sube.

ERROR 11 (Sesión 4) — obj.castShadow = false blanket después del traverse
  Causaba que los castShadow aplicados desde LIGHT_VALUES[idx][3] fueran
  inmediatamente sobreescritos a false.
  Solución: Eliminar esa línea. Cada rama (useDefaults / !useDefaults)
  maneja castShadow de forma independiente.

ERROR 12 (Sesión 4) — shadowMapSize hardcodeado a 2048 en togBtn
  Causaba que el slider "Tamaño Mapa" no tuviera efecto real al activar
  sombras desde el panel.
  Solución: Usar _shadowMapSize global en lugar de literal 2048.

LUCES.glb:
  - 22 nodos → Three.js pierde los nombres de nodo al cargar
  - Solución: array LIGHT_NAMES hardcodeado en el mismo orden del GLB
  - Carga via _loadLucesGLB(url, onDone, useDefaults) — soporta recarga
  - ~7KB — SÍ se embebe en .galeria (base64 ~9KB)
  - Exportar desde Blender: ✅ Selected Objects ✅ Punctual Lights ❌ Geometry
  - useDefaults=true  → usa LIGHT_VALUES + LIGHT_NAMES hardcodeados
  - useDefaults=false → lee nombre y escala intensidad (* 0.00005) del GLB

CARGA DE LUCES EN RUNTIME:
  _loadLucesGLB(url, onDone, useDefaults=true):
    - Elimina gltf.scene anterior si existe (_lucesGltfScene)
    - Limpia sceneLights[]
    - Traversea nuevas luces
    - useDefaults=true: aplica LIGHT_VALUES[idx] → intensity, color,
      distance, castShadow + shadow config completa si castShadow=true
    - useDefaults=false: escala intensidad raw * 0.00005, lee nombre del nodo
    - Llama onDone() para reconstruir panel
  Botón CARGAR LUCES.glb:
    - file picker → file.arrayBuffer() → guarda en _lucesGLBBuffer
    - createObjectURL → _loadLucesGLB(url, cb, false)
  Botón ELIMINAR LUCES:
    - scene.remove(_lucesGltfScene) + limpiar panel + shadowMap.enabled=false
